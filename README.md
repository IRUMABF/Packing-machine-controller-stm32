# Контролер упаковочного станка для спайок (банок фарби)

**Платформа:** STM32 (F4 серія)  
**Мова програмування:** C / C++ (STM32CubeIDE, PlatformIO)  
**Версія прошивки:** 1  

## Огляд

Цей проект реалізує програмне забезпечення для керування автоматичною лінією пакування **спайок** (лінійок з 6 з'єднаних маленьких банок). Станок виконує три основні операції:

1. **Закривання (запресування) кришок** на кожній спайці.
2. **Накопичення 4 спайок** на пакувальному конвеєрі за допомогою соленоїдного упору та планки зсуву.
3. **Пакування 4 спайок** у поліетиленовий пакет (підготовка пакета, заштовхування, скидання).

Крім того, система контролює наявність пакетів на складі та автоматично ставиться на паузу при їх відсутності.

## Особливості

- **Багатозадачність без RTOS** – всі затримки неблокуючі (таймери `millis()`), що дозволяє паралельно виконувати три задачі:
  - Задача A – закривання кришок з ігноруванням 5 імпульсів датчика (оскільки спайка має 6 банок).
  - Задача B – накопичення 4 спайок і пакування з використанням OUTPUT2 (планка зсуву) та OUTPUT8 (соленоїдний упор).
  - Задача C – моніторинг датчика наявності пакетів (переривання) з індикацією LED2 та постановкою на паузу.
- **Підтримка кнопок START/STOP** з антибонсом.
- **Індикація стану**: LED1 (зелений) – робота/пауза, LED2 (червоний/жовтий) – відсутність пакетів.
- **Керування кроковими двигунами** (драйвери з напрямком та кроком) та пневматичними клапанами (8 виходів).
- **Легке налаштування таймів** через файл `config.h`.

## Схема підключення (основні сигнали)

| Вхід/Вихід | Призначення | Пін STM32 (приклад) |
|------------|-------------|----------------------|
| `BTN_START` | Кнопка пуску | PA0 |
| `BTN_STOP`  | Кнопка стоп (NC) | PA1 |
| `SENSOR1`   | Датчик банок на закриванні | PA2 |
| `SENSOR2`   | Датчик спайки на пакуванні | PA3 |
| `SENSOR3`   | Датчик наявності пакетів | PA4 (переривання) |
| `OUTPUT1`   | Закривання кришки | PB0 |
| `OUTPUT2`   | Планка зсуву спайок | PB1 |
| `OUTPUT3`   | Штовхач у пакет | PB2 |
| `OUTPUT4`   | Вертикаль платформи (вниз) | PB3 |
| `OUTPUT5`   | Горизонталь платформи (вперед) | PB4 |
| `OUTPUT6`   | Скидання пакета | PB5 |
| `OUTPUT7`   | Вакуум | PB6 |
| `OUTPUT8`   | Соленоїдний упор | PB7 |
| `MOTOR1_STEP` | Крок для двигуна 1 | PA8 |
| `MOTOR1_DIR`  | Напрямок двигуна 1 | PA9 |
| `MOTOR2_STEP` | Крок для двигуна 2 | PA10 |
| `MOTOR2_DIR`  | Напрямок двигуна 2 | PA11 |
| `LED1` | Зелена лампа | PA12 |
| `LED2` | Червона лампа (немає пакетів) | PA13 |

## Алгоритм роботи (стисло)

### Задача A (закривання)
1. Конвеєр 1 рухається постійно.
2. При спрацюванні `SENSOR1`:
   - якщо це **перша банка** в спайці → зупинити двигун, активувати `OUTPUT1` на час `CLOSE_CAP_HOLD_TIME_MS`, потім запустити двигун;
   - інакше (наступні 5 банок) → просто збільшити лічильник, не зупиняючи двигун.
3. Після 6-ї банки лічильник скидається, і наступна спайка знову викликає зупинку.

### Задача B (накопичення 4 спайок та пакування)
1. Конвеєр 2 рухається постійно.
2. Соленоїд `OUTPUT8` чергує стан для кожної нової спайки (LOW – для непарної, HIGH – для парної), змінюючи позицію упору.
3. Коли `SENSOR2` фіксує спайку, активується `OUTPUT2` (планка зсуву) на час `SHIFT_BOTTLES_TIME_MS`, спайка зсувається в бік.
4. Після зсуву:
   - якщо це **не 4-та спайка** → `OUTPUT2` вимикається, `OUTPUT8` перемикається, конвеєр продовжує рух;
   - якщо це **4-та спайка** → `OUTPUT2` залишається висунутим, запускається цикл пакування.
5. **Цикл пакування**:
   - підготовка пакета (опускання платформи, вакуум, підйом, переміщення вперед, відкриття пакета);
   - висування `OUTPUT3` (штовхач) – 4 спайки заштовхуються в пакет;
   - одночасне повернення `OUTPUT2` та `OUTPUT3`;
   - вимкнення вакууму, імпульс скидання пакета `OUTPUT6`;
   - повернення платформи додому;
   - обнулення лічильників, повторення циклу.

### Задача C (контроль пакетів)
- Працює на перериванні по `SENSOR3`.
- Коли `SENSOR3 = LOW` (пакети закінчились):
  - вмикається `LED2`;
  - якщо пакування не виконується → негайна пауза;
  - якщо пакування виконується → встановлюється прапорець, і пауза відбудеться після його завершення.
- Коли `SENSOR3 = HIGH` (пакети з'явились) → `LED2` гасне, але станок залишається на паузі до натискання `START`.
